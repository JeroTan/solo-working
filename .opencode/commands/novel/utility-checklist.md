# User Input: $ARGUMENTS

## Objective

Run a predefined checklist against the provided text or the current state of the story to ensure no critical elements have been forgotten.

## Execution Steps

### 1. Identify Target

Determine what the user wants to check (e.g., Chapter 1, or the overall plot).

### 2. Execute Checklist

Run the following checks based on the target:

**Pre-Write Checklist (for new chapters)**
- Is the character motivation clear for this scene?
- Does the setting have at least two sensory details?
- Does the scene advance the plot or develop the character?
- Is there conflict (internal or external)?

**Post-Write Checklist (for finished chapters)**
- Did the chapter accomplish the goal set in the plan?
- Is the pacing appropriate?
- Is the dialogue natural and subtext-driven?
- Did I tell instead of show?

**General Story Checklist**
- Are all planted foreshadowing elements tracked?
- Are the character arcs progressing as planned?

### 3. Output Report

Provide a brief report highlighting any missed items or areas for improvement.
